### Note on the committed Firebase identifiers
`lib/firebase_options.dart` contains the real `projectId` (`habitview-1574c`),
`appId`, and `messagingSenderId`. These are **identifiers, not secrets** — they
identify the Firebase project but do not grant access on their own (access is
gated by Firebase Auth + Security Rules + App Check). The actual access-relevant
value, the Android API key, is a placeholder. This is the standard, safe posture
for an open-source Flutter app. **Enable Firebase App Check** before public
release to prevent project-id abuse (tracked in `RELEASE_CHECKLIST.md`).
